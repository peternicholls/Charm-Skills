# Security Policy

## Supported Versions

Security updates apply to the latest released version of the Skill suite.

## Reporting A Vulnerability

If you find a security issue in the Skills or repository automation, report it privately to the maintainers using GitHub private vulnerability reporting when enabled. If private reporting is not enabled, open a minimal public issue that says a vulnerability exists and ask for a private contact path. Do not include exploit details, secrets, or sensitive examples in a public issue.

## Scope

Security-relevant issues include:

- Guidance that would leak secrets in logs, screenshots, videos, or terminal output.
- Guidance that encourages unsafe SSH authentication or authorization defaults.
- Repository automation that exposes tokens or release credentials.
- Malicious or unsafe scripts added to the repository.

## Non-Scope

General Charm library vulnerabilities should be reported to the relevant upstream project.
